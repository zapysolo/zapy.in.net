## Why Astro?

After building this portfolio as a single `index.html` file, I wanted something more maintainable — reusable components, type-safe content, and proper SEO metadata without the overhead of a full framework.

Astro gives you all of that with zero JavaScript shipped by default.

## What Changed

- Sections became proper Astro components
- Project and skill data moved to typed `.ts` files
- Blog posts and project details use Content Collections with Zod schema validation
- SEO meta tags, Open Graph, and sitemap are now handled centrally in `BaseLayout.astro`

## Content Collections

The most useful Astro feature for a portfolio is Content Collections. Define a schema once in `src/content/config.ts` and every blog post or project detail page is validated at build time.

```ts
const blogCollection = defineCollection({
  schema: z.object({
    title: z.string(),
    date: z.date(),
    tags: z.array(z.string()),
  }),
});
```

If you add a post missing a `title`, the build fails — not your users.
